## 전체 흐름 요약

Ansible로 k3s 클러스터를 프로비저닝하고, GitHub Actions로 Docker 이미지를 빌드·푸시하며, ArgoCD GitOps로 **FE·BE·infra 매니페스트**를 자동 배포·관리하는 구조. Supabase(ai/data)·모니터링·Longhorn 등은 **Helm으로 직접 설치**돼 ArgoCD 관리 밖이다. ML 파이프라인은 로컬에서 수동 실행해 벡터 데이터를 생성한다.

```
개발자 git push
  → GitHub Actions (CI): 빌드 → GHCR 이미지 푸시 → manifest 태그 업데이트
  → ArgoCD (CD): Git 변경 감지 → k3s 클러스터 자동 sync

개발자 로컬 (수동)
  → 4K_ML/generate_vectors/generate_vectors.py 실행
  → 외부 Supabase SaaS에서 scene 데이터 수집 → 시계열 처리 → vm4 movie_vectors upsert
```

## `4K_BE/` — 백엔드 (FastAPI)

K3s `be` 네임스페이스에 배포되는 FastAPI 서버. 외부 노출 없이 클러스터 내부 DNS(`backend.be.svc.cluster.local:8000`)로만 접근.

| 파일/경로                   | 역할                                                                                                                                                                                    |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Dockerfile`                | `python:3.11-slim` 기반. requirements 설치 후 app/ 복사                                                                                                                                 |
| `requirements.txt`          | `fastapi`, `uvicorn[standard]`, `httpx`, `python-dotenv`                                                                                                                                |
| `app/main.py`               | FastAPI 앱. `GET /health`, `GET /api/movies?page=N` (TMDB 목록 + Supabase in_db 체크), `POST /api/movies/{tmdb_id}` (TMDB 상세 fetch → Supabase upsert), `DELETE /api/movies/{tmdb_id}` |
| `DB_SCRIPTS/seed_movies.py` | TMDB API에서 영화 메타데이터를 가져와 Supabase `movies` 테이블에 upsert. 배치 50개 단위                                                                                                 |
| `DB_SCRIPTS/.env`           | `TMDB_API_KEY`, `DATA_SUPABASE_URL`, `DATA_SUPABASE_KEY` 보관 (gitignore)                                                                                                               |

> 테이블 DDL(`movies`, `movie_vectors`)과 RPC 함수(`find_preferred_movies` 등)는 **Studio SQL Editor에서 수동 실행**했으며, 현재 저장소에 `.sql` 파일로 보관돼 있지 않다. BE 배포는 `4k-be-secrets` Secret(`envFrom`)을 참조하므로 ns에 미리 생성돼 있어야 한다.

## `4K_ML/` — ML 파이프라인 (오프라인 배치)

클라이맥스 벡터를 생성해 vm4 Supabase `movie_vectors` 테이블에 저장하는 오프라인 배치 스크립트. CI/CD 없이 로컬에서 수동 실행.

| 파일/경로                              | 역할                                                                                                                                                    |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `requirements.txt`                     | `httpx`, `python-dotenv`, `numpy`, `scipy`                                                                                                              |
| `.env`                                 | `EXT_SUPABASE_URL`, `EXT_SUPABASE_KEY` (외부 SaaS), `DATA_SUPABASE_URL`, `DATA_SUPABASE_KEY`, `DATA_BASIC_USER/PASS` (vm4 nginx Basic Auth) — gitignore |
| `generate_vectors/generate_vectors.py` | 메인 스크립트. 외부 SaaS `training` 스키마에서 `scene_scores·scenes·subtitles` fetch → Python join → 시계열 처리 → vm4 `movie_vectors` upsert           |

### 시계열 처리 상세

```
(progress_ratio, score) 포인트 수집
  ↓ progress_ratio 기준 정렬
  ↓ 200포인트 선형 리샘플링 (progress_ratio 범위 내)
  ↓ Savitzky-Golay 스무딩 (window=11, polyorder=2)
  ↓ z-score 정규화 (mean=0, std=1)
  → 200차원 float 벡터
```

- `MIN_SCENES=5` 미만이거나 분산=0인 영화는 스킵
- upsert 충돌 키: `(tmdb_id, vector_version)`

## `Ansible/values/` — Helm values 파일

각 서비스를 `helm install/upgrade`할 때 참조하는 values. `helm-values/`(ingress-nginx·Longhorn 초기 설치용)와 용도가 같으며, 디렉토리만 분리돼 있다.

> ⚠️ Supabase(ai/data)·모니터링·Redis는 **Helm으로 직접 설치**된 상태이며 ArgoCD 관리 대상이 아니다. 따라서 이 values 변경은 자동 sync되지 않고 `helm upgrade`로 수동 반영해야 한다. ArgoCD가 sync하는 것은 `manifests/4k-fe`, `manifests/4k-be`, `manifests/apps`뿐이다.

| ------------------------ | ----------------------------------------------------------- |
