# KakaoCloud_Project 디렉토리 구조

## 전체 흐름 요약

Ansible로 k3s 클러스터를 프로비저닝하고, GitHub Actions로 Docker 이미지를 빌드·푸시하며, ArgoCD GitOps로 Supabase·FE·모니터링 등 전체 서비스를 자동 배포·관리하는 구조.

```
개발자 git push
  → GitHub Actions (CI): 빌드 → GHCR 이미지 푸시 → manifest 태그 업데이트
  → ArgoCD (CD): Git 변경 감지 → k3s 클러스터 자동 sync
```

---

## `.github/` — GitHub Actions CI

| 하위 경로 | 역할 |
|---|---|
| `workflows/deploy-4k-fe.yml` | `4K_FE/**` 변경 시 Docker 빌드 → GHCR 푸시 → `Ansible/manifests/4k-cinema/kustomization.yaml` 이미지 태그 자동 업데이트 |

---

## `4K_FE/` — 프론트엔드 (Next.js 16)

4K Cinema 영화 추천 서비스의 프론트엔드. App Router 기반, 클라이맥스 그래프 시각화.

| 파일/경로 | 역할 |
|---|---|
| `Dockerfile` | multi-stage 빌드. deps → builder → runner(standalone) 3단계로 이미지 최소화 |
| `.dockerignore` | node_modules, .next, .env 등 빌드 불필요 파일 제외 |
| `next.config.ts` | `output: 'standalone'` 설정 (독립 실행 가능한 서버 번들 생성) |
| `app/` | App Router 소스. `page.tsx`(온보딩), `dashboard/page.tsx`(메인), `components/`, `lib/` |

---

## `Ansible/` — 인프라 프로비저닝 + K3s 매니페스트

k3s 클러스터 구성·운영을 위한 Ansible 자동화 코드와 ArgoCD가 관리하는 모든 K3s 매니페스트를 함께 관리.

| 하위 경로 | 역할 |
|---|---|
| `playbooks/` | 클러스터 초기 셋업 플레이북. `00` ping 확인 → `01` 공통 설정(swap 비활성화, 커널 모듈) → `02` k3s master 설치 → `03` k3s agent 설치 → `04` Longhorn 스토리지 전제조건 설치 순으로 실행 |
| `manifests/apps/` | ArgoCD `infra-manifests` Application이 sync. ingress-nginx 설정, cert-manager ClusterIssuer, Grafana Studio Ingress 등 **인프라 레벨** 매니페스트 |
| `manifests/4k-cinema/` | ArgoCD `4k-cinema` Application이 sync. Kustomize 기반 FE 앱 매니페스트 (Deployment, Service, Ingress, kustomization.yaml). GitHub Actions CI가 `kustomization.yaml`의 `newTag`를 git SHA로 자동 갱신 |
| `manifests/argocd/` | ArgoCD Application 오브젝트 정의. 최초 1회 `kubectl apply`로 등록 후 ArgoCD가 나머지를 자동 관리 |
| `manifests/test/` | GPU 동작 확인, nginx 동작 테스트용 임시 매니페스트 |
| `helm-values/` | ingress-nginx, Longhorn Helm 설치에 사용하는 values 파일 |
| `generate-apply-secrets/` | AI/Data 서비스의 k3s Secret을 `.env`에서 생성·적용하는 쉘 스크립트 |
| `group_vars/all.yaml` | 모든 호스트에 공통 적용되는 Ansible 변수 |
| `inventory.ini` | 관리 대상 노드 목록 (master/agent 구분) |
| `ansible.cfg` | Ansible 전역 설정 |
| `kubeconfig`, `k3s_node_token`, `ssh_config` | 클러스터 접속 자격증명 및 SSH 설정 (gitignore 대상) |

---

## `Ansible/values/` — Helm values 파일 (ArgoCD GitOps)

ArgoCD가 Git에서 직접 읽는 각 서비스의 Helm values. `Ansible/helm-values/`(Ansible 초기 구축용)와 구분되며, 이쪽은 ArgoCD가 매 sync마다 참조한다.

| 파일 | 역할 |
|---|---|
| `values-argocd.yaml` | ArgoCD 자체 설정 (ingress, insecure 모드 등) |
| `values-ai.yaml` | Supabase AI 서비스 설정 (JWT/DB Secret ref) |
| `values-data.yaml` | 데이터 관련 서비스 설정 |
| `values-prometheus.yaml` | Prometheus 모니터링 설정 |
| `values-loki.yaml` | Loki 로그 수집 설정 |
| `values-promtail.yaml` | Promtail 로그 에이전트 설정 |
| `values-redis.yaml` | Redis 캐시 설정 |
