## 전체 흐름 요약

Ansible로 k3s 클러스터를 프로비저닝하고, GitHub Actions로 Docker 이미지를 빌드·푸시하며, ArgoCD GitOps로 Supabase·FE·BE·모니터링 등 전체 서비스를 자동 배포·관리하는 구조.

```
개발자 git push
  → GitHub Actions (CI): 빌드 → GHCR 이미지 푸시 → manifest 태그 업데이트
  → ArgoCD (CD): Git 변경 감지 → k3s 클러스터 자동 sync
```

## `4K_FE/` — 프론트엔드 (Next.js)

4K Cinema 영화 추천 서비스의 프론트엔드. App Router 기반, Supabase에서 실시간 데이터를 fetch해 표시.

| 파일/경로 | 역할 |
|---|---|
| `Dockerfile` | multi-stage 빌드. deps → builder → runner(standalone) 3단계로 이미지 최소화 |
| `.dockerignore` | node_modules, .next, .env 등 빌드 불필요 파일 제외 |
| `next.config.ts` | `output: 'standalone'` 설정 |
| `app/page.tsx` | 온보딩/랜딩 페이지 |
| `app/dashboard/page.tsx` | 메인 대시보드. Supabase REST API에서 영화 목록 fetch, 연도·장르 필터, 무한 스크롤(PAGE_SIZE=120), 최근 본 영화(localStorage) 섹션 |
| `app/movie_list/page.tsx` | 매니저 페이지. TMDB 최신 영화 목록(페이지네이션), Supabase DB 존재 여부 배지, 추가/삭제 버튼 |
| `app/api/manager/movies/route.ts` | 서버사이드 proxy. GET: TMDB 목록 + DB in_db 상태 조회, POST: 영화 추가 (BE로 프록시) |
| `app/api/manager/movies/[tmdb_id]/route.ts` | 서버사이드 proxy. DELETE: 영화 삭제 (BE로 프록시) |
| `app/lib/data.ts` | `Movie` 인터페이스, `Filters` 타입, Supabase URL/anon key 상수, `posterUrl()`·`genreList()`·`castList()` 유틸, `getRecentIds()`·`addRecentId()` localStorage 헬퍼 |
| `app/components/PosterCard.tsx` | TMDB CDN 포스터 이미지, 3D tilt 인터랙션, glare 효과 |
| `app/components/DetailOverlay.tsx` | 영화 상세 오버레이. overview, 감독, 배우, YouTube 트레일러 iframe embed |
| `app/components/FilterBar.tsx` | 연도 듀얼 핸들 슬라이더(데이터 최솟값~현재 연도) + EditableYear 화살표 스텝퍼, 장르·상황 칩 필터 |
| `app/components/RandomModal.tsx` | 랜덤 추천 모달. 전달받은 `movies[]`에서 무작위 선택 + 셔플 애니메이션 |
| `app/components/MiniGraph.tsx` | SVG 기반 미니 클라이맥스 그래프 (Tutorial 데모용) |
| `app/components/Tutorial.tsx` | 최초 방문자 온보딩 (4단계, localStorage로 완료 여부 관리) |

## `4K_BE/` — 백엔드 (FastAPI)

K3s `be` 네임스페이스에 배포되는 FastAPI 서버. 외부 노출 없이 클러스터 내부 DNS(`backend.be.svc.cluster.local:8000`)로만 접근.

| 파일/경로 | 역할 |
|---|---|
| `Dockerfile` | `python:3.11-slim` 기반. requirements 설치 후 app/ 및 DB_SCRIPTS/.env 복사 |
| `requirements.txt` | `fastapi`, `uvicorn[standard]`, `httpx`, `python-dotenv` |
| `app/main.py` | FastAPI 앱. `GET /health`, `GET /api/movies?page=N` (TMDB 목록 + Supabase in_db 체크), `POST /api/movies/{tmdb_id}` (영화 추가), `DELETE /api/movies/{tmdb_id}` (영화 삭제) |
| `DB_SCRIPTS/create_service_tables.sql` | `public.movies`·`public.movie_vectors` 테이블 DDL. pgvector extension, BIGSERIAL PK, tmdb_id UNIQUE 인덱스 |
| `DB_SCRIPTS/seed_movies.py` | TMDB API에서 403편 메타데이터를 가져와 Supabase `data` ns movies 테이블에 upsert. 배치 50개 단위 |
| `DB_SCRIPTS/.env` | `TMDB_API_KEY`, `DATA_SUPABASE_URL`, `DATA_SUPABASE_KEY` 보관 (gitignore) |

## `Ansible/` — 인프라 프로비저닝 + K8s 매니페스트

k3s 클러스터 구성·운영을 위한 Ansible 자동화 코드와 ArgoCD가 관리하는 모든 K8s 매니페스트를 함께 관리.

| 하위 경로 | 역할 |
|---|---|
| `playbooks/` | 클러스터 초기 셋업 플레이북. `00` ping → `01` 공통 설정 → `02` k3s master → `03` k3s agent → `04` Longhorn 전제조건 순서로 실행 |
| `manifests/apps/` | ArgoCD `infra-manifests` Application이 sync. cert-manager ClusterIssuer, ingress-nginx ConfigMap, Studio/API/Grafana/ArgoCD Ingress |
| `manifests/4k-cinema/` | ArgoCD `4k-cinema` Application이 sync. Kustomize 기반 FE 앱 매니페스트. CI가 `kustomization.yaml`의 `newTag`를 git SHA로 자동 갱신 |
| `manifests/4k-be/` | ArgoCD `4k-be` Application이 sync. Kustomize 기반 BE 앱 매니페스트. CI가 `kustomization.yaml`의 `newTag`를 git SHA로 자동 갱신 |
| `manifests/argocd/` | ArgoCD Application 오브젝트 정의. 최초 1회 `kubectl apply`로 등록 |
| `manifests/test/` | GPU 동작 확인, nginx 테스트용 임시 매니페스트 |
| `helm-values/` | ingress-nginx, Longhorn Helm 초기 설치에 사용 (1회성) |
| `generate-apply-secrets/` | AI/Data Supabase의 k3s Secret을 `.env`에서 생성·적용하는 쉘 스크립트 |
| `group_vars/all.yaml` | 모든 호스트에 공통 적용되는 Ansible 변수 |
| `inventory.ini` | 관리 대상 노드 목록 (master/agent 구분) |
| `kubeconfig`, `k3s_node_token`, `ssh_config` | 클러스터 접속 자격증명 및 SSH 설정 (gitignore 대상) |

## `Ansible/values/` — Helm values 파일 (ArgoCD GitOps)

ArgoCD가 Git에서 직접 읽는 각 서비스의 Helm values. `helm-values/`(초기 1회 설치용)와 구분되며, 이 디렉토리는 ArgoCD가 매 sync마다 참조한다.

| 파일 | 역할 |
|---|---|
| `values-argocd.yaml` | ArgoCD 자체 설정 (ingress, insecure 모드 등) |
| `values-ai.yaml` | Supabase AI ns 설정 (db, studio, kong, storage, minio 활성) |
| `values-data.yaml` | Supabase data ns 설정 (db, studio, kong, rest, meta 활성) |
| `values-prometheus.yaml` | Prometheus + Grafana + Alertmanager 설정 |
| `values-loki.yaml` | Loki 로그 저장소 설정 |
| `values-promtail.yaml` | Promtail 로그 수집 에이전트 설정 |
| `values-redis.yaml` | Redis 캐시 설정 |

## `Ansible/manifests/apps/ingress-studio.yaml` — Ingress 구조

한 파일에 여러 Ingress가 정의됨. ArgoCD `infra-manifests`가 자동 적용. 모두 `letsencrypt-prod` 사용.

| Ingress 이름 | 호스트 | 경로 | 백엔드 | 인증 |
|---|---|---|---|---|
| `supabase-data-studio` | data.4kakao.kro.kr | `/*` | Studio:3000 | Basic Auth |
| `supabase-data-api` | data.4kakao.kro.kr | `/rest/v1`, `/auth/v1`, `/storage/v1` | Kong:8000 | Supabase JWT |
| `supabase-ai-studio` | ai.4kakao.kro.kr | `/*` | Studio:3000 | Basic Auth |
| `grafana` | grafana.4kakao.kro.kr | `/*` | Grafana:80 | Grafana 자체 인증 |
| `argocd` | argocd.4kakao.kro.kr | `/*` | argocd-server:80 | ArgoCD 자체 인증 |

4K Cinema FE Ingress (`Ansible/manifests/4k-cinema/ingress.yaml`):

| 항목 | 값 |
|---|---|
| 호스트 | cinema.4kakao.kro.kr |
| TLS | letsencrypt-prod |
| ssl-redirect | true (HTTP → HTTPS 자동 리다이렉트) |
