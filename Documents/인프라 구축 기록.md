## 2. 네트워크 구성

### 트래픽 흐름

```
[인바운드 HTTPS]
사용자 브라우저
  → DNS (4kakao.kro.kr → 210.109.83.10)
  → vm1 Public IP:443
  → Ingress-Nginx Pod (vm1, hostNetwork=true)
  → cert-manager TLS termination
  → Studio/Grafana/ArgoCD Service (ClusterIP)
  → 해당 Pod

[아웃바운드]
vm2/3/4/5 (Private)
  → vm1 NAT/MASQUERADE
  → 인터넷
```

### vm1 NAT 설정

- `net.ipv4.ip_forward = 1`
- iptables MASQUERADE: `10.1.0.0/16 → enp3s0`
- KakaoCloud 콘솔: Source/Dest Check 비활성화, Private 라우팅 default → vm1

### TLS-SAN (K3s API)

- `--tls-san 210.109.83.10`
- `--tls-san 10.1.1.10`

## 4. K3s 설치 옵션

### Server (vm1)

```
--disable traefik
--write-kubeconfig-mode 644
--tls-san 210.109.83.10
--tls-san 10.1.1.10
--advertise-address 10.1.1.10
--node-ip 10.1.1.10
```

### Agent (vm2~5)

```
K3S_URL=https://10.1.1.10:6443
K3S_TOKEN=<node-token>
--node-ip <private-ip>
```

## 5. Ingress-Nginx

### 설치 옵션 (Helm)

- vm1에만 배포 (`workload=ingress` 라벨)
- `hostNetwork: true` → vm1의 80/443 직접 점유
- Master 노드 toleration 설정
- ServiceLB 미사용

### 로그 포맷 (JSON)

ConfigMap `ingress-nginx-controller`로 access log를 JSON으로 변경. Loki에서 `| json` 파이프로 필드별 검색 가능.

## 6. GPU Operator

### 구성

- NVIDIA Driver 580.126.20
- CUDA Runtime 13.0
- Tesla T4 인식 (`nvidia.com/gpu: 1`)
- containerd 통합 (K3s 경로: `/var/lib/rancher/k3s/agent/etc/containerd/`)

### 주요 트러블슈팅

- **containerd config 깨짐**: GPU Operator의 toolkit이 config 수정 중 RuntimeService 다운 → `config.toml*` 삭제 + k3s-agent 재시작으로 복구
- **GPU 리소스 미등록**: device-plugin 재시작으로 해결

---

## 7. Longhorn (스토리지)

### 구성

- vm4에만 배포 (`workload=data`)
- `defaultReplicaCount: 1` (단일 노드, 복제 없음)
- `local-path`가 default StorageClass, `longhorn`은 명시 지정

### 주요 트러블슈팅

- **uninstall 실패**: `deleting-confirmation-flag` 필요 → `--no-hooks`로 우회 후 수동 정리
- **engineimage 사라짐**: CRD 강제 삭제 시 vm4 호스트의 engine 바이너리도 같이 사라짐 → 전체 재설치
- **backuptarget "default" 없음**: 강제 정리 후 누락 → 수동 BackupTarget 리소스 생성으로 해결

## 9. Supabase 구성

### 배포 방식

- Helm repo: `https://supabase-community.github.io/supabase-kubernetes` (chart v0.5.6)
- ArgoCD multi-source로 관리: Helm repo(차트) + GitHub repo(values 파일) 조합
- ArgoCD Application: `argocd-app-supabase-ai.yaml`, `argocd-app-supabase-data.yaml`

### data ns (서비스 DB)

- 활성: db, studio, kong, rest, meta, functions
- 비활성: auth, realtime, storage, imgproxy, analytics, vector, minio
- pgvector extension 활성화 (벡터 검색용)
- 메타데이터 + 도파민 시계열 + 임베딩 모두 PG에 저장
- 검색: pgvector cosine + DTW 재순위

### ai ns (학습 DB)

- 활성: db, studio, kong, rest, meta, functions, storage, imgproxy, minio
- Storage는 체크포인트 저장용 (MinIO 백엔드)
- pgvector extension 활성화
- 비활성: auth, realtime, analytics, vector

### Secret 관리

- `Ansible/generate-apply-secrets/generate-secrets.sh` 스크립트로 일괄 생성
- `secrets-data/.env`, `secrets-ai/.env`에 평문 보관 (gitignore)
- 생성되는 secret:
  - `supabase-jwt`, `supabase-db`, `supabase-dashboard`, `supabase-smtp`, `supabase-analytics`, `supabase-s3` (ai만), `supabase-minio` (ai만)

## 16. CI/CD 파이프라인 (4K Cinema FE)

```
git push (4K_FE/** 변경)
  → GitHub Actions (.github/workflows/deploy-4k-fe.yml)
      ① Docker 빌드 (Next.js standalone, multi-stage)
      ② ghcr.io/sanggyoon/4k-cinema:<sha> GHCR 푸시
      ③ Ansible/manifests/4k-cinema/kustomization.yaml 이미지 태그 커밋 [skip ci]
  → ArgoCD (git 변경 감지, ~3분 폴링)
      ④ kubectl apply -k Ansible/manifests/4k-cinema/ 자동 실행
      ⑤ fe 네임스페이스 Pod 롤링 업데이트
```

- **이미지 레지스트리**: `ghcr.io/sanggyoon/4k-cinema` (GHCR, `GITHUB_TOKEN` 사용)
- **태그 전략**: git short SHA (7자) + `latest`
- **무한루프 방지**: manifest 업데이트 커밋에 `[skip ci]` 태그
- **ArgoCD 설정**: `prune: true`, `selfHeal: true`

### 초기 등록 (최초 1회)

```bash
# GHCR 패키지를 Public으로 설정 (GitHub → Packages → 4k-cinema → Settings)
# 또는 k3s 노드에 GHCR imagePullSecret 추가

# ArgoCD Application 등록
kubectl apply -f Ansible/manifests/argocd/argocd-app-4k-cinema.yaml
```

## 18. 트러블슈팅 체크리스트

| 증상                     | 확인사항                                             |
| ------------------------ | ---------------------------------------------------- |
| `kubectl` 연결 실패      | KUBECONFIG 환경변수, TLS-SAN, SG 6443                |
| Private VM 인터넷 안 됨  | vm1 NAT 규칙, ip_forward, Source/Dest Check          |
| Pod이 노드에 안 떠짐     | nodeSelector 라벨, taint/toleration                  |
| GPU 인식 안 됨           | nvidia-device-plugin 재시작, containerd config       |
| Longhorn 설치 실패       | open-iscsi, iscsid, CRD/webhook 잔여물, BackupTarget |
| kubelet 502              | 노드 SG 10250, kubelet 동작 여부                     |
| Supabase Pod ConfigError | Secret 키 이름 mismatch (특히 minio, s3)             |
| 인증서 발급 실패         | DNS 전파, rate limit, 80 포트 접근                   |
| Ingress 충돌             | 같은 host 중복 사용                                  |
| ArgoCD sync 안 됨        | Git push 여부, repoURL/path 정확성                   |
