# 인프라 구축 방법 (명령어)

> 처음부터 클러스터를 재구성할 때 순서대로 실행하면 됨.  
> 모든 명령어는 로컬(Mac) 기준. `KUBECONFIG` 설정 이후 `kubectl` 명령어 사용 가능.

---

## 0. 사전 준비

```bash
# Ansible 설치 (로컬)
pip install ansible

# Helm 설치 (로컬)
brew install helm

# ssh_config 적용
cp Ansible/ssh_config ~/.ssh/config   # 또는 Include로 추가

# kubeconfig 준비 (Step 1 완료 후 vm1에서 복사)
export KUBECONFIG=./Ansible/kubeconfig
```

---

## 1. Ansible: 클러스터 프로비저닝

```bash
cd Ansible

# 0) 연결 확인
ansible-playbook playbooks/00-ping.yml

# 1) 공통 설정 (swap off, 커널 모듈, sysctl)
ansible-playbook playbooks/01-common.yml

# 2) k3s master (vm1)
ansible-playbook playbooks/02-k3s-server.yml

# 3) k3s agent (vm2~5)
ansible-playbook playbooks/03-k3s-agent.yml

# 4) Longhorn 전제조건 (vm4: open-iscsi, iscsid)
ansible-playbook playbooks/04-longhorn-prereq.yml
```

### kubeconfig 가져오기

```bash
scp vm1:/etc/rancher/k3s/k3s.yaml Ansible/kubeconfig
# 파일 내 server 주소를 Public IP로 변경
sed -i '' 's/127.0.0.1/210.109.83.10/g' Ansible/kubeconfig
export KUBECONFIG=$(pwd)/Ansible/kubeconfig
```

### node-token 저장

```bash
ssh vm1 "sudo cat /var/lib/rancher/k3s/server/node-token" > Ansible/k3s_node_token
```

## 3. vm1 NAT 설정 (vm2~5 인터넷 아웃바운드)

```bash
# vm1에서 실행
ssh vm1 "sudo sysctl -w net.ipv4.ip_forward=1"
ssh vm1 "sudo iptables -t nat -A POSTROUTING -s 10.1.0.0/16 -o enp3s0 -j MASQUERADE"

# 영구 적용
ssh vm1 "echo 'net.ipv4.ip_forward = 1' | sudo tee -a /etc/sysctl.conf"
ssh vm1 "sudo apt install -y iptables-persistent && sudo netfilter-persistent save"
```

> KakaoCloud 콘솔에서 Source/Dest Check 비활성화 및 Private 서브넷 라우팅 default → vm1 설정 필요.

## 11. Supabase Secrets 생성 및 적용

### 네임스페이스 생성

```bash
kubectl create namespace ai
kubectl create namespace data
```

### Secret 생성

```bash
cd Ansible/generate-apply-secrets

# data 인스턴스
./generate-secrets.sh data

# ai 인스턴스
./generate-secrets.sh ai
```

### Secret 적용

```bash
# data
cd secrets-data && ./apply-secrets.sh && cd ..

# ai
cd secrets-ai && ./apply-secrets.sh && cd ..
```

### Ingress Basic Auth Secret 생성 (Studio 로그인용)

```bash
# data ns
PASS=$(grep DASHBOARD_PASSWORD secrets-data/.env | cut -d= -f2)
kubectl create secret generic supabase-dashboard-basic \
  --namespace=data \
  --from-literal=auth="$(htpasswd -nb supabase $PASS)"

# ai ns
PASS=$(grep DASHBOARD_PASSWORD secrets-ai/.env | cut -d= -f2)
kubectl create secret generic supabase-dashboard-basic \
  --namespace=ai \
  --from-literal=auth="$(htpasswd -nb supabase $PASS)"
```

## 13. Service DB 초기화 (movies 테이블)

> Supabase data ns가 정상 기동된 후 1회 실행.

### 테이블 생성 + RLS 비활성화

Supabase Studio (`https://data.4kakao.kro.kr`) → SQL Editor에서 실행:

```sql
-- 4K_BE/DB_SCRIPTS/create_service_tables.sql 내용 전체 붙여넣기
-- (pgvector extension, movies 테이블, movie_vectors 테이블 생성)

-- 자가호스팅 Supabase에서 service_role JWT가 RLS 자동 우회 안 함 → 수동 비활성화
ALTER TABLE movies DISABLE ROW LEVEL SECURITY;
```

### TMDB 메타데이터 시드

```bash
# .env 설정 확인 (TMDB_API_KEY, DATA_SUPABASE_KEY)
cat 4K_BE/DB_SCRIPTS/.env

# 의존성 설치
pip install httpx python-dotenv

# 실행 (403편 TMDB 메타데이터 → data.movies 테이블 upsert)
python 4K_BE/DB_SCRIPTS/seed_movies.py
```

> 처음 1개를 삽입하고 DB에서 확인한 뒤 나머지를 배치(50개씩) 처리함.  
> Supabase REST API는 Kong(8000)을 거쳐야 하므로 `supabase-data-api` Ingress가 먼저 적용되어 있어야 함.

## 15. 4K Cinema FE 배포 (ArgoCD + GitHub Actions)

### GHCR 이미지 접근 설정

GitHub 패키지가 **Public**이면 별도 설정 불필요.  
**Private**인 경우 k3s 노드에 imagePullSecret 추가:

```bash
# GitHub Personal Access Token (read:packages 권한 필요)
kubectl create secret docker-registry ghcr-secret \
  --namespace fe \
  --docker-server=ghcr.io \
  --docker-username=sanggyoon \
  --docker-password=<GITHUB_PAT>

# deployment.yaml에 추가 후 git push (ArgoCD가 자동 반영)
# spec.template.spec.imagePullSecrets:
#   - name: ghcr-secret
```

### ArgoCD Application 등록 (최초 1회)

```bash
kubectl apply -f Ansible/manifests/argocd/argocd-app-4k-cinema.yaml

# 등록 확인
kubectl get application 4k-cinema -n argocd
```

### 이후 배포 흐름

```bash
# 4K_FE/ 코드 수정 후 push하면 자동 배포
git add 4K_FE/
git commit -m "feat: ..."
git push

# GitHub Actions가 이미지 빌드 → GHCR push → kustomization.yaml 태그 업데이트
# ArgoCD가 ~3분 내 자동 sync → Pod 롤링 업데이트
```

### FE Pod 상태 확인

```bash
kubectl get pods -n fe
kubectl logs -n fe -l app=frontend --tail=50
kubectl get ingress -n fe
kubectl get certificate -n fe
```
