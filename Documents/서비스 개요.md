## 1. 서비스 소개

**4K Cinema (Peakly)** 는 영화의 긴장감·감동 흐름을 시각화한 **클라이맥스 그래프**를 기반으로 영화를 추천하는 서비스입니다.

장르나 평점이 아닌 *"영화를 보면서 어떤 감정 곡선을 경험하고 싶은가"* 를 기준으로 영화를 필터링하고 탐색할 수 있습니다.

- **서비스 URL**: https://cinema.4kakao.kro.kr
- **데이터**: TMDB API에서 수집한 403편 메타데이터 + Supabase PostgreSQL 저장
- **이미지**: TMDB CDN (`image.tmdb.org`)

## 2. 주요 기능

| 기능 | 설명 |
|---|---|
| 영화 목록 탐색 | 포스터 카드 그리드, 무한 스크롤 (PAGE_SIZE=120) |
| 연도 필터 | 듀얼 핸들 슬라이더 + 화살표 스텝퍼로 제작 연도 범위 설정 |
| 장르 필터 | 액션, 공포, 코미디, 애니메이션 등 칩 형태 다중 선택 |
| 상황 필터 | "혼자 볼 때", "연인과", "가족과" 등 시청 상황별 분류 |
| 클라이맥스 그래프 | 영화별 긴장감·감동 흐름 SVG 시각화 |
| 영화 상세 오버레이 | 포스터, 감독, 배우, overview, YouTube 트레일러 embed |
| 랜덤 추천 | 현재 필터 조건 내에서 무작위 영화 추천 + 셔플 애니메이션 |
| 최근 본 영화 | localStorage 기반, 대시보드 상단 섹션에 표시 |
| 온보딩 튜토리얼 | 최초 방문자 대상 4단계 튜토리얼 (완료 여부 localStorage 관리) |

## 3. 화면 구성

### 랜딩/온보딩 페이지 (`/`)

- 서비스 진입점
- 최초 방문자는 자동으로 튜토리얼 시작
- `Tutorial.tsx`: 클라이맥스 그래프 사용법, 필터, 랜덤 추천 설명 (4단계)

### 대시보드 (`/dashboard`)

- 메인 화면. 영화 포스터 카드 그리드
- 좌측: `FilterBar` (연도 슬라이더, 장르·상황 칩)
- 상단: 최근 본 영화 섹션
- 우상단: 랜덤 추천 버튼 (`RandomModal`)
- 카드 클릭 → `DetailOverlay` (영화 상세 + 트레일러)

### 매니저 페이지 (`/movie_list`)

- 관리자용 TMDB 영화 관리 화면
- TMDB 최신 영화 목록 + DB 저장 여부 배지 표시
- 영화 추가(POST) / 삭제(DELETE) 버튼
- FE API Route → BE FastAPI → Supabase 순으로 프록시

## 4. 기술 스택

### Frontend (`4K_FE/`)

| 항목 | 내용 |
|---|---|
| 프레임워크 | Next.js 16, App Router |
| 언어 | TypeScript |
| 스타일 | CSS Modules + globals.css |
| 상태 | React state (서버 컴포넌트 + 클라이언트 컴포넌트 혼용) |
| 데이터 fetch | Supabase REST API (클라이언트), Next.js API Route (서버사이드 프록시) |
| 배포 | `output: standalone` → multi-stage Docker → GHCR |

### Backend (`4K_BE/`)

| 항목 | 내용 |
|---|---|
| 프레임워크 | FastAPI |
| 런타임 | Python 3.11, Uvicorn |
| 데이터베이스 | Supabase (PostgreSQL + pgvector) |
| 외부 API | TMDB API (영화 메타데이터, 이미지, 트레일러) |
| 네트워크 | ClusterIP only — 외부 미노출, FE에서 내부 DNS로만 호출 |

### 데이터베이스 스키마

```sql
-- public.movies (서비스 핵심 테이블)
CREATE TABLE movies (
  id           BIGSERIAL PRIMARY KEY,
  tmdb_id      INTEGER UNIQUE NOT NULL,
  title        TEXT,
  overview     TEXT,
  poster_path  TEXT,
  release_date TEXT,
  genre_ids    INTEGER[],
  vote_average NUMERIC,
  -- 클라이맥스 그래프 관련 컬럼 등
  ...
);

-- public.movie_vectors (pgvector 임베딩)
CREATE TABLE movie_vectors (
  id       BIGSERIAL PRIMARY KEY,
  movie_id BIGINT REFERENCES movies(id),
  embedding vector(1536)
);
```

## 5. 데이터 흐름

### 사용자 영화 탐색

```
브라우저
  → dashboard/page.tsx (Supabase REST API 직접 fetch)
  → data.4kakao.kro.kr/rest/v1/movies
  → Kong (API Gateway) → PostgREST → PostgreSQL
```

### 영화 추가/삭제 (매니저)

```
브라우저
  → /api/manager/movies (Next.js API Route — 서버사이드)
  → http://backend.be.svc.cluster.local:8000 (K3s 내부 DNS)
  → FastAPI (4K_BE)
  → Supabase data ns PostgreSQL
```

### TMDB 트레일러

```
DetailOverlay.tsx
  → TMDB API (클라이언트 직접 호출)
  → YouTube iframe embed
```

## 6. 접근 정보

| 항목 | 값 |
|---|---|
| 서비스 URL | https://cinema.4kakao.kro.kr |
| 인증 | 없음 (공개 서비스) |
| BE 내부 URL | http://backend.be.svc.cluster.local:8000 |
| K8s namespace (FE) | `fe` |
| K8s namespace (BE) | `be` |
| GHCR 이미지 (FE) | `ghcr.io/sanggyoon/4k-cinema:<sha>` |
| GHCR 이미지 (BE) | `ghcr.io/sanggyoon/4k-be:<sha>` |

### Pod 상태 확인

```bash
# FE
kubectl get pods -n fe
kubectl logs -n fe -l app=frontend --tail=50

# BE
kubectl get pods -n be
kubectl logs -n be -l app=backend --tail=50

# 인증서 상태
kubectl get certificate -n fe
```
