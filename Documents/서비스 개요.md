## 1. 서비스 소개

**4K Cinema (Peakly)** 는 영화의 긴장감·감동 흐름을 시각화한 **클라이맥스 그래프**를 기반으로 영화를 추천하는 서비스입니다.

장르나 평점이 아닌 *"영화를 보면서 어떤 감정 곡선을 경험하고 싶은가"* 를 기준으로 영화를 필터링하고 탐색할 수 있습니다.

- **서비스 URL**: https://cinema.4kakao.kro.kr
- **데이터**: TMDB API에서 수집한 영화 메타데이터 + Supabase PostgreSQL 저장
- **이미지**: TMDB CDN (`image.tmdb.org`)

## 2. 주요 기능

| 기능 | 설명 |
|---|---|
| 영화 목록 탐색 | 포스터 카드 그리드, 무한 스크롤 (PAGE_SIZE=120) |
| 연도 필터 | 듀얼 핸들 슬라이더 + 화살표 스텝퍼로 제작 연도 범위 설정 |
| 장르 필터 | 액션, 공포, 코미디, 애니메이션 등 칩 형태 선택 |
| 클라이맥스 그래프 | 영화별 긴장감·감동 흐름 SVG 시각화 (벡터 있는 영화만) |
| 유사 영화 추천 | pgvector 코사인 유사도 기반 비슷한 패턴 영화 4편 |
| 영화 상세 오버레이 | 포스터, 감독, 배우, overview, YouTube 트레일러 embed |
| 랜덤 추천 | 현재 필터 조건 내에서 무작위 영화 추천 + 셔플 애니메이션 |
| 선호/비선호 | 카드별 좋아요/싫어요 토글, 비선호 영화 목록에서 제외 |
| 최근 본 영화 | localStorage 기반, 대시보드 상단 가로 스크롤 섹션 |
| 온보딩 튜토리얼 | 최초 방문자 대상 4단계 튜토리얼 (완료 여부 localStorage 관리) |

## 3. 화면 구성

### 랜딩/온보딩 페이지 (`/`)

- 서비스 진입점. WebGL 배경 애니메이션(`BackgroundThread`)
- 기능 소개 카드 3개 + "시작하기" CTA 버튼

### 대시보드 (`/dashboard`)

- 메인 화면. 영화 포스터 카드 그리드 (무한 스크롤)
- 상단: 최근 본 영화 가로 스크롤 섹션
- 헤더: 제목 검색바, 필터 버튼(`FilterBar`), 랜덤 추천 버튼(`RandomModal`)
- 카드 클릭 → `DetailOverlay` (영화 상세 + 클라이맥스 그래프 + 유사 영화)

### 매니저 페이지 (`/movie_list`)

- 관리자용 TMDB 영화 관리 화면
- TMDB 최신 영화 목록 + DB 저장 여부 배지 표시
- 영화 추가(POST) / 삭제(DELETE) 버튼
- FE API Route → BE FastAPI → Supabase 순으로 프록시

## 4. 기술 스택

### Frontend (`4K_FE/`)

| 항목 | 내용 |
|---|---|
| 프레임워크 | Next.js (App Router) |
| 언어 | TypeScript |
| 스타일 | globals.css (Tailwind 없음, 인라인 스타일 + CSS 클래스 혼용) |
| 데이터 fetch | Supabase REST API 직접 호출 (클라이언트), Next.js API Route (서버사이드 BE 프록시) |
| 배포 | `output: standalone` → multi-stage Docker → GHCR |

### Backend (`4K_BE/`)

| 항목 | 내용 |
|---|---|
| 프레임워크 | FastAPI |
| 런타임 | Python 3.11, Uvicorn |
| 역할 | TMDB API 프록시 + Supabase movies 테이블 CRUD |
| 네트워크 | ClusterIP only — 외부 미노출, FE Next.js API Route에서 내부 DNS로만 호출 |

### ML 파이프라인 (`4K_ML/`)

| 항목 | 내용 |
|---|---|
| 실행 방식 | 오프라인 배치 (수동 실행) |
| 언어 | Python 3.11 |
| 라이브러리 | httpx, numpy, scipy (savgol_filter) |
| 역할 | 외부 Supabase SaaS `training` 스키마에서 scene_scores 수집 → 시계열 처리 → vm4 `movie_vectors` upsert |
| 시계열 처리 | ① 200포인트 선형 리샘플링 ② Savitzky-Golay 스무딩 ③ z-score 정규화 |

### 데이터베이스 스키마 (vm4 Supabase)

```sql
-- service.movies
CREATE TABLE movies (
    id             BIGSERIAL PRIMARY KEY,
    tmdb_id        BIGINT NOT NULL UNIQUE,
    imdb_id        VARCHAR(30),
    title          VARCHAR(255),
    original_title VARCHAR(255),
    poster_path    VARCHAR(255),
    director       VARCHAR(255),
    release_year   INT,
    runtime        INT,
    genre          VARCHAR(255),   -- "Action, Drama" 형식
    actors         VARCHAR(500),   -- "Actor1, Actor2" 형식 (상위 5명)
    overview       TEXT,
    youtube_key    VARCHAR(50),
    has_vector     BOOLEAN DEFAULT false,  -- movie_vectors 존재 여부 (트리거 자동 관리)
    created_at     TIMESTAMPTZ DEFAULT NOW()
);

-- service.movie_vectors  (pgvector)
CREATE TABLE movie_vectors (
    id               BIGSERIAL PRIMARY KEY,
    tmdb_id          BIGINT NOT NULL REFERENCES movies(tmdb_id) ON DELETE CASCADE,
    vector           VECTOR(200) NOT NULL,   -- 200차원 클라이맥스 벡터
    vector_version   VARCHAR(50),            -- "rule-v1"
    normalization    VARCHAR(50),            -- "zscore"
    smoothing_method VARCHAR(50),            -- "savgol_w11_p2"
    created_at       TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (tmdb_id, vector_version)
);
```

## 5. 데이터 흐름

### ML 벡터 생성 (오프라인 배치)

```
외부 Supabase SaaS (training 스키마)
  subtitles(tmdb_id) + scenes(progress_ratio) + scene_scores(score)
      ↓ Python join → tmdb_id별 (progress_ratio, score) 시계열 수집
      ↓ 200pt 리샘플링 → Savitzky-Golay 스무딩 → z-score 정규화
      ↓ upsert
vm4 Supabase — movie_vectors (tmdb_id, vector[200])
      ↓ INSERT 트리거
movies.has_vector = true 자동 갱신
```

### 영화 추가 (매니저)

```
브라우저 → /api/manager/movies (Next.js API Route)
  → http://backend.be.svc.cluster.local:8000/api/movies/{tmdb_id} (K3s 내부)
  → FastAPI: TMDB API에서 상세정보 fetch (감독, 배우, 트레일러 포함)
  → vm4 Supabase movies 테이블 upsert
```

### 사용자 영화 탐색

```
브라우저 → dashboard/page.tsx
  → data.4kakao.kro.kr/rest/v1/movies (Supabase REST 직접)
  → 포스터 카드 그리드 렌더링 (has_vector 포함)
```

### 영화 상세 (클라이맥스 그래프 + 유사 영화)

```
카드 클릭 → DetailOverlay
  ├── fetchVector(tmdb_id)
  │     → movie_vectors?tmdb_id=eq.{tmdb_id}
  │     → ClimaxGraph SVG 렌더링
  └── fetchSimilarMovies(movie.id)
        → Supabase RPC find_similar_movies (pgvector 코사인 유사도)
        → 비슷한 패턴 영화 4편
```
