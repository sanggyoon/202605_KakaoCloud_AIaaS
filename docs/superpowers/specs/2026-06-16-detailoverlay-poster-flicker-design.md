# DetailOverlay 상단 포스터 + 깜빡임 수정 설계

**작성일:** 2026-06-16
**범위:** `4K_FE/app/components/DetailOverlay.tsx` (FE만).

---

## 1. 상단 포스터 + 정보 2단

- 현재 단일 컬럼(포스터 없음)인 상단(연도·러닝타임 / 제목 / 원제 / 장르)을 **2단**으로:
  - **좌: 포스터** — `image.tmdb.org/t/p/w342{poster_path}`, 너비 ~200px, `aspect-ratio: 2/3`, 라운드, 없으면 🎬 플레이스홀더.
  - **우: 기존 정보 블록**(연도·러닝타임 / 제목 / 원제 / 장르) 그대로.
- 좁은 화면(`flex-wrap`)에선 세로 스택. 그 아래 CLIMAX CURVE·유사추천·시놉시스·감독/배우·트레일러는 불변.
- `posterUrl`은 w500 고정이라, 상세 포스터용은 인라인으로 w342 URL 구성(또는 별도 헬퍼). 상세는 w342면 충분.

---

## 2. 깜빡임 수정

- **원인(유력):** 컨테이너 `background: rgba(0,0,0,0.88)` + `backdropFilter: blur(14px)` + `overflow: auto`. backdrop-filter가 걸린 스크롤 컨테이너는 Chrome에서 스크롤/리페인트 시 깜빡이고, 뒤의 애니메이션 BackgroundThread를 매 프레임 재샘플링하며 악화.
- **수정:** `backdropFilter` 제거 + 배경을 거의 불투명 다크(`rgba(8,9,13,0.97)`)로. 뒤 캔버스 의존이 사라져 깜빡임 해소. 시각적으론 동일한 어두운 오버레이.
- fadeIn 애니메이션은 마운트 1회라 유지(재마운트 토글 없음 — 확인됨).

---

## 3. 테스트
- `npm run build` + 수동(상단 포스터/정보 2단, 스크롤 시 깜빡임 없음, 좁은 화면 스택).
