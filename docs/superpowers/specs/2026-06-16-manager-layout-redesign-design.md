# 매니저 페이지 레이아웃 재구성 설계

**작성일:** 2026-06-16
**범위:** `4K_FE/app/manager/page.tsx` **레이아웃만** 재구성(가독성). 로직·엔드포인트·데이터 흐름 변경 없음.
**근거:** 사용자 제공 와이어프레임. 비율·글자 크기·간격은 적절히 다듬음.

---

## 1. 목표

기존 요소(방문자·기간조회·처리현황·바로가기·활성모델·수집 컨트롤·진행 로그)를 와이어프레임 배치로 재배열해 가독성을 높인다. **새 기능 없음**(기간조회 결과에 선택 날짜 표기만 추가).

---

## 2. 섹션 배치 (위→아래)

1. **헤더** — 기존 "서비스 모니터링" 유지.
2. **방문자 행**: 누적/30일/7일/하루 `StatCard` 4개 + 우측 **컴팩트 기간조회 박스**(시작 `date` / 종료 `date` / 조회 버튼 / 결과). 한 줄 그리드. (기존 별도 기간 UI를 이 줄로 통합)
3. **2단 패널**: 좌 **처리 현황**(리스트형 단계칩 + 간략 설명 1줄) | 우 **바로가기**(리스트형 링크 + 간략 설명 1줄). 반응형 2열(좁으면 1열).
4. **활성 모델**: 좌 큰 **모델 버전** 카드 + 우 **2×2 지표**(arousal MAE / valence MAE / arousal Spearman / valence Spearman). arousal 2개 accent 강조.
5. **영화 메타 수집 카드**: 설명("tmdb 인기도 순으로 신규 영화 메타 수집") + 수량입력 + 수집버튼, 그 아래 **진행 로그 패널**(상시 표시; 잡 없으면 "대기 중").
6. **자막 수집 카드**: 설명("subdl에서 자막 없는 영화의 자막 수집") + 수량입력 + 수집버튼 + **진행 로그 패널**(상시).

---

## 3. 변경 상세

- **기간조회 결과에 선택 날짜 표시**: `2026-06-01 ~ 2026-06-16 · 방문 N명` 형태로 표기(선택 날짜가 보이게).
- **JobBanner 위치 이동**: 기존 하단 공통 배너 → 각 수집 카드 내부 로그 영역으로. 잡 없을 땐 옅은 "대기 중" 플레이스홀더.
- **처리 현황**: 현재 단계칩(자막/파싱/라벨/스코어/벡터 + 값 칩) 유지하되 2단 좌측에 배치 + 상단에 한 줄 설명("자막→파싱→라벨→스코어→벡터 단계별 처리 수").
- **바로가기**: 영화 정보 리스트(내부) + 외부 5개(Grafana/ArgoCD/ArgoWorkflow/SVC DB/AI DB)를 **리스트형**(세로 목록)로, 각 항목에 짧은 설명.
- 스타일: 카드/그리드/색은 기존 `StatCard`·`cardGrid`·`sectionTitle`·`actionBtn`·`numInput`·`hintText` 재사용 + 2단/2×2용 그리드 추가. 비율·폰트는 가독성 위주로 조정.

---

## 4. 범위 밖 / 불변

- BE·프록시·데이터 로직, 수집/방문자/모델 동작, 라우팅 전부 불변.
- 모바일 완벽 대응은 목표 아님(좁을 때 1열 폴백 정도).

## 5. 테스트

- `npm run build`(타입체크) + 배포 후 수동(배치·기간조회 날짜표시·로그 위치).
