# Peakly 튜토리얼 재설계 — 설계 문서

- 날짜: 2026-06-17
- 대상 컴포넌트: `4K_FE/app/components/Tutorial.tsx` (전면 재작성)
- 관련 와이어링: `4K_FE/app/dashboard/page.tsx`

## 1. 목표와 배경

현재 튜토리얼은 **프로토타입 버전**으로, 다음과 같이 실제 서비스와 어긋난다.

- 서비스명이 `4K Cinema`로 표기됨 (현재는 **Peakly**, 태그라인 "한 편의 감정을 선으로 그리다").
- 4단계 구성이며 순서가 필터 → 랜덤 → 그래프로, 핵심 컨셉인 클라이맥스 그래프가 마지막에 나온다.
- 그래프 설명에 **분위기(valence, 색) 축이 빠져 있다.** 실제 그래프는 `높이=고조(arousal)` + `색=분위기(valence)` 두 축이다.
- 필터 설명이 "연도·장르"로만 뭉뚱그려져 있고, 실제 필터(제목검색/연도범위/선호·비선호 장르/선호·비선호 영화)를 반영하지 못한다.
- step 1~2에서 헤더를 spotlight로 강조하는 로직이 있으나 새 구성에선 불필요.

**이번 작업은 "새로 만든다"는 전제로 문구·디자인·단계 구성을 전면 교체한다.**

## 2. 전체 형식 (확정)

**A안: 중앙 모달 캐러셀 + 단계별 "살아있는 데모"**

- 화면을 어둡게 깐 배경(backdrop) 위, **가운데 고정폭 카드(약 340~360px)**에서 한 단계씩 진행한다.
- 카드는 세로로 쌓이는 단일 컬럼 구조라 **모바일/데스크탑 모두 안정적**이다. (가로 분할형 풀스크린 C안은 모바일에서 무너져 제외.)
- 카드 안 **데모 영역(약 150px 높이)**에 단계별 시각 데모(그려지는 그래프, 필터 토글, 포스터 셔플 등)를 넣어 몰입감을 확보한다 — C안의 비주얼 강점을 흡수.
- 진행 표시는 **점(dots)** + `스킵` / `다음`(마지막은 `시작하기`) 버튼.

**비고:** 현재의 헤더 spotlight(clipPath, `isHeaderHighlighted`) 로직은 제거한다. 모든 단계가 순수 중앙 모달이다.

## 3. 단계 구성 (5단계, 확정)

각 단계는 `{ label, title, desc, demo }`로 구성된다. 데모는 **자체 완결형 프레젠테이션 비주얼**이며 라이브 데이터를 fetch하지 않는다(아래 §5 참조).

### STEP 1 / 5 — 환영
- **title:** `Peakly에 오신 걸 환영합니다`
- **desc:** 한 줄 가치 제안. 예) "영화의 감정 흐름을 선으로 그려, 당신의 클라이맥스에 맞는 영화를 찾아드려요."
- **demo:** Peakly 로고(글로우) + 태그라인 "한 편의 감정을 선으로 그리다" 등장 애니메이션.

### STEP 2 / 5 — 클라이맥스 그래프 (핵심)
- **title:** `감정을 선으로 읽다`
- **desc:** 두 축을 명확히 전달. "**높이는 감정의 고조**, **색은 분위기**를 나타냅니다. 어두운 분위기(teal)에서 밝은 분위기(pink)까지."
- **demo:** 좌→우로 **그려지는 클라이맥스 곡선**(teal→purple→pink 그라데이션). `↑ 높이 = 고조`, `색 = 분위기` 주석. 가능하면 호버 십자선 힌트.
- **데이터 두 축 매핑(정확성 기준):**
  - arousal → 곡선 높이
  - valence → 곡선 색 (teal 부정/어두움 → purple 중립 → pink 긍정/밝음), 영화별 정규화. (`app/lib/color.ts`의 팔레트 A와 일치)

### STEP 3 / 5 — 필터링
- **title:** `원하는 조건으로 좁히기`
- **desc:** 실제 필터를 모두 언급. "제목 검색, 연도 범위, 선호/비선호 장르, 선호/비선호 영화로 추천 풀을 좁힙니다."
- **demo:** 미니 필터 데모 — 검색바(🔍 영화 제목…) + 연도 듀얼 핸들 슬라이더 + 장르 pill(선호=accent, 비선호=red) 토글 + 영화 칩. 순차 토글 애니메이션.
- **반영 대상(실제 `FilterBar.tsx`):** 제목검색 / 연도범위(1900~현재, 듀얼 핸들) / 선호 장르(단일, accent) / 비선호 장르(다중, red) / 선호 영화 칩(accent) / 비선호 영화 칩.

### STEP 4 / 5 — 랜덤픽
- **title:** `고민될 땐 랜덤픽`
- **desc:** "무엇을 볼지 모르겠다면, 전체 DB에서 무작위로 한 편을 골라드려요."
- **demo:** 포스터 여러 장이 **셔플되다 한 장으로 확정**되는 애니메이션. (실제 `RandomModal`의 셔플 연출 톤과 일치)

### STEP 5 / 5 — 상세 모달 (클라이맥스 커브 상세)
- **title:** `더 깊이: 클라이맥스 커브`
- **desc:** "포스터를 누르면 전체 감정 곡선과 **비슷한 패턴의 영화**까지 볼 수 있어요."
- **demo:** 실제 `DetailOverlay` 모습 축약 반영 —
  - 헤더 `CLIMAX CURVE` + 우측 `자막·장면 AI 분석`
  - 촘촘한 굴곡 곡선(valence 색 흐름)
  - **분위기 범례 바** (`어두운 분위기` ↔ `밝은 분위기`, teal→pink 그라데이션)
  - `비슷한 패턴의 영화 · 클라이맥스 유사도 기반` — 큰 매치%(예: 48%) + 작은 포스터 + 제목/메타("2023 · 액션 · 코미디 · 중반 정점의 산형 곡선") + 미니 곡선 + `›` 행 2개
- **마지막 버튼:** `시작하기` → 튜토리얼 종료.

## 4. 컴포넌트 아키텍처

`Tutorial.tsx`를 다음 구조로 재작성한다.

- **`Tutorial`** (기존 props 유지: `step, onNext, onSkip, onComplete`)
  - 배경 backdrop(단일, clipPath 없음) + 중앙 카드 렌더.
  - `STEPS` 배열에서 현재 단계의 `label/title/desc`를 그리고, `step`에 따라 데모 컴포넌트를 스위치.
  - 진행 dots, 스킵/다음(`시작하기`) 버튼.
- **데모 서브컴포넌트** (같은 파일 내 또는 `Tutorial/` 하위로 분리):
  - `DemoWelcome` — 로고+태그라인
  - `DemoGraph` — 그려지는 곡선 (재사용: `MiniGraph`가 `valence` 색을 지원하므로 활용 가능, 또는 전용 SVG)
  - `DemoFilters` — 미니 필터 토글
  - `DemoRandom` — 포스터 셔플
  - `DemoDetail` — 커브 + 범례 + 유사영화 행
- **재사용 자산:** `MiniGraph`(arousal+valence 색), `app/lib/color.ts`의 `valenceColorAt`/`valenceGradientStops`, `app/lib/svgPath.ts`의 `catmullRomPath`.

각 데모는 입력 props 없이 자체 완결적이며, 시각 표현만 담당한다(단일 책임).

## 5. 데이터 정책

- 데모는 **라이브 데이터를 fetch하지 않는다.** 단계 전환·네트워크 지연으로 튜토리얼이 깨지면 안 되므로, **고정 더미 데이터/정적 SVG**로 구성한다. (현재 코드도 `DEMO_GRAPH` 상수 사용 — 동일 원칙 유지.)
- 더미 곡선 데이터는 실제 곡선 느낌(촘촘한 굴곡)을 살린 상수로 정의한다.

## 6. 디자인 토큰 / 비주얼

기존 Peakly 톤을 그대로 따른다.

- 배경 backdrop: `rgba(0,0,0,0.72)`
- 카드: `linear-gradient(160deg, rgba(22,18,40,.98), rgba(10,9,18,.98))`, border `rgba(123,97,255,.22)`, radius 18, 보라 글로우 그림자.
- accent: `var(--accent)` (#7b61ff). 비선호=red 계열(#ff7070).
- valence 그라데이션: teal `#2dd4bf` → purple `#7b61ff` → pink `#ff6ec7`.
- 라벨: 10px/800/letter-spacing .2em, accent 색.
- 버튼: accent 배경 + 검정 텍스트, 보라 글로우.

## 7. 와이어링 변경 (`dashboard/page.tsx`)

- `STEPS.length`가 4→5가 되므로 `onComplete` 트리거 조건(마지막 step)은 컴포넌트 내부에서 `step === STEPS.length - 1`로 자동 처리(현행 유지).
- `isHeaderHighlighted` / 헤더 spotlight 관련 로직 **제거**.
- **localStorage 키 변경:** `4k_tutorial_done` → **`peakly_tutorial_done`**. 의도: 재설계된 튜토리얼을 기존 사용자에게도 한 번 더 노출. (기존 키 마이그레이션은 하지 않음.)
- 헤더의 튜토리얼 재시작 버튼(`onClick={() => setTutorialStep(0)}`)은 유지.

## 8. 인터랙션

- `스킵` → `onSkip` (즉시 종료 + 완료 기록).
- `다음` → `onNext` (step+1). 마지막 단계는 `시작하기` → `onComplete`.
- dots는 현재 진행도 표시(클릭 이동은 비포함 — YAGNI).
- 키보드: 기존에 없으므로 이번 범위에서 추가하지 않음.

## 9. 범위 밖 (Non-goals)

- 단계 직접 점프(dots 클릭 이동), 키보드 내비게이션.
- 실제 데이터 연동 데모, 다국어.
- DetailOverlay/FilterBar/RandomModal 등 **실제 기능 컴포넌트 수정** (튜토리얼은 이들을 설명만 함).
- 매니저 모드 안내.

## 10. 검증 기준

- 5단계가 순서대로 렌더되고 문구가 §3과 일치.
- 모바일 폭(≤375px)에서 카드가 넘치지 않고 데모가 깨지지 않음.
- `스킵`/`시작하기`/완료 후 새로고침 시 `peakly_tutorial_done`로 인해 재노출되지 않음.
- 헤더 재시작 버튼으로 step 0부터 다시 시작됨.
- 그래프 데모 색이 valence 팔레트(teal→purple→pink)와 일치.
- `4K_FE` 빌드/타입체크 통과 (`AGENTS.md` 준수: Next.js 가이드 확인 후 작업).
