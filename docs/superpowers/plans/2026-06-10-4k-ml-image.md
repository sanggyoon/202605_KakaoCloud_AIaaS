> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 4K_ML을 CUDA 이미지로 컨테이너화(CI로 ghcr 푸시)하고, 자막 파싱(C)을 vm5 GPU에서 도는 Argo WorkflowTemplate로 실행한다.

**Architecture:** `pytorch/pytorch` CUDA 베이스 Dockerfile(all-MiniLM bake) + GitHub Actions CI(BE 패턴 미러) + `subtitle-parse` WorkflowTemplate(ai ns, GPU) + ArgoCD app. DB는 내부 kong 서비스로 접속.

**Tech Stack:** Docker(pytorch CUDA), GitHub Actions, Argo Workflows, ArgoCD, k8s(nvidia GPU). 코드/유닛테스트 없음 — Dockerfile/YAML 산출물 + CI·클러스터 검증.

**Spec:** `docs/superpowers/specs/2026-06-10-4k-ml-image-design.md`

**검증 방법 주의:** 로컬에 Docker 데몬이 없어 이미지 빌드는 **CI(푸시 시)**가 수행한다. 매니페스트/CI YAML은 로컬에서 유효성만 확인하고, 실제 빌드·Sync·워크플로 실행은 Task 5(사용자 핸드오프)에서 검증한다.

## Task 2: GitHub Actions CI

**Files:**
- Create: `.github/workflows/deploy-4k-ml.yml`

- [ ] **Step 1: 워크플로 작성** (BE 패턴 미러, 태그 bump 대상은 WorkflowTemplate)

`.github/workflows/deploy-4k-ml.yml`:

```yaml
name: Deploy 4K ML

on:
  push:
    branches:
      - main
    paths:
      - '4K_ML/**'
      - '.github/workflows/deploy-4k-ml.yml'

env:
  IMAGE: ghcr.io/sanggyoon/4k-ml

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      packages: write

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Extract short SHA
        id: vars
        run: echo "sha=$(echo ${{ github.sha }} | cut -c1-7)" >> $GITHUB_OUTPUT

      - name: Build and push Docker image
        uses: docker/build-push-action@v6
        with:
          context: ./4K_ML
          push: true
          tags: |
            ${{ env.IMAGE }}:${{ steps.vars.outputs.sha }}
            ${{ env.IMAGE }}:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Update image tag in WorkflowTemplate
        run: |
          sed -i 's|image: ghcr.io/sanggyoon/4k-ml:.*|image: ghcr.io/sanggyoon/4k-ml:${{ steps.vars.outputs.sha }}|' \
            Ansible/manifests/4k-ml/workflowtemplate-subtitle-parse.yaml

      - name: Commit and push manifest update
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add Ansible/manifests/4k-ml/workflowtemplate-subtitle-parse.yaml
          git commit -m "ci: update 4k-ml image → ${{ steps.vars.outputs.sha }} [skip ci]"
          git pull --rebase
          git push
```

## Task 3: subtitle-parse WorkflowTemplate (vm5 GPU)

**Files:**
- Create: `Ansible/manifests/4k-ml/workflowtemplate-subtitle-parse.yaml`

- [ ] **Step 1: WorkflowTemplate 작성**

`Ansible/manifests/4k-ml/workflowtemplate-subtitle-parse.yaml`:

```yaml
# 자막 파싱(C)을 vm5 GPU에서 실행. 제출: argo submit --from workflowtemplate/subtitle-parse -n ai
apiVersion: argoproj.io/v1alpha1
kind: WorkflowTemplate
metadata:
  name: subtitle-parse
  namespace: ai
spec:
  serviceAccountName: argo-workflow
  podSpecPatch: |
    runtimeClassName: nvidia
  entrypoint: main
  templates:
    - name: main
      nodeSelector:
        workload: gpu
      container:
        image: ghcr.io/sanggyoon/4k-ml:latest   # CI가 sha로 bump
        command: ["python", "-m", "subtitle_parse.parse_subtitles"]
        envFrom:
          - secretRef:
              name: 4k-ml-secrets
        resources:
          limits:
            nvidia.com/gpu: 1
```
